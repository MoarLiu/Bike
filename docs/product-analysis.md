# 产品分析：本地优先大纲与脑图工具

## 背景和目标

本项目参考幕布的核心体验：用大纲组织信息，再从同一份层级数据切换出思维导图、演示、导出等形态。产品目标不是复刻幕布的云端协作平台，而是做一个功能和逻辑相近、数据默认保存在本机、可通过 iCloud Drive 备份和迁移的本地优先工具。

可参考的公开来源：

- [幕布官网首页](https://mubu.com/home)：强调大纲笔记、思维导图、在线协作和多端同步。
- [幕布关于页](https://mubu.com/about)：定位为结构化思考、学习、写作和任务管理工具。
- [幕布帮助中心](https://mubu.com/help)：包含编辑、分享协作、导入导出、快捷键等使用说明。
- [幕布导入导出帮助](https://mubu.com/help/articles/pc/advanced_skills/de546d9a568748b38707b5242fe9c76f/)：说明 PC/网页端支持的导入导出格式，包括 OPML、FreeMind、XMind 等迁移方向。

## 幕布的核心产品逻辑

幕布的产品内核可以抽象为一棵“可编辑、可折叠、可变换视图的树”。

1. 信息以大纲节点为基本单位
   - 每个文档由主题节点组成。
   - 节点天然有父子层级，适合表达标题、分论点、任务拆解、会议议程、读书笔记。
   - 用户的主要操作不是富文本排版，而是增删节点、调整层级、折叠展开、聚焦局部。

2. 大纲是主输入，脑图是同源视图
   - 大纲模式适合快速录入和整理。
   - 思维导图模式把同一棵树映射成视觉化结构，适合展示、复盘和发散。
   - 这类产品的关键是“同一份结构，多种表现”，避免大纲和脑图各维护一套数据。

3. 结构化内容可继续延展
   - 节点可以承载备注、任务状态、标签、图片、附件、链接等扩展信息。
   - 文档层可以支持搜索、分类、收藏、回收站、历史版本、导入导出。
   - 云端版本会进一步加入多端同步、分享、协作、权限、团队空间等能力。

4. 快捷键驱动高频编辑
   - Enter 创建同级主题。
   - Tab / Shift+Tab 调整层级。
   - 上下移动、折叠展开、聚焦主题等操作需要尽量少打断输入流。
   - 这决定了编辑器必须先优化键盘体验，再补充鼠标工具栏。

## 主要功能模块

### 1. 文档与工作区

幕布类产品通常以“工作区/文件夹/文档列表”承载内容管理。本项目可以先采用轻量工作区：

- 工作区保存所有本地文档和当前打开文档。
- 文档列表支持新建、重命名、复制、删除、搜索。
- 文档元数据包括标题、创建时间、更新时间、节点数量。
- 后续可加入文件夹、收藏、最近编辑、回收站，但首版不必做复杂空间管理。

### 2. 大纲编辑器

这是核心模块，决定产品是否好用。

必要能力：

- 创建同级节点和子节点。
- 修改节点文本。
- 调整节点层级。
- 上移/下移节点。
- 删除节点。
- 折叠/展开子树。
- 聚焦某个节点，只编辑其子树。
- 节点备注。
- 任务勾选。
- 节点颜色或标记。

建议后续增强：

- 多选节点和批量移动。
- 拖拽排序。
- 撤销/重做。
- Markdown 快捷输入。
- 图片、链接、附件或本地文件引用。
- 节点级创建时间、更新时间和完成时间。

### 3. 思维导图视图

脑图不应成为第二套编辑器。更合理的实现是：

- 从文档树实时生成图形布局。
- 支持选中节点、居中定位、展开/收起。
- 首版允许只读或轻编辑。
- 后续再支持拖拽换父级、主题样式、导出图片。

可替代范围：

- 首版用简化树状脑图代替复杂商业脑图引擎。
- 不必实现幕布完整主题模板、连线样式、画布缩放、自由布局。
- 只要能让用户从结构上看清整篇文档，就满足本地版核心目标。

### 4. 演示视图

幕布类工具很适合把层级大纲直接变成演示提纲。本项目可以把演示视图做成轻量阅读模式：

- 文档标题作为封面。
- 一级节点作为章节。
- 子节点作为要点。
- 支持聚焦节点后只演示局部内容。

可替代范围：

- 首版不做复杂幻灯片编辑器。
- 不做 PPT 动画、母版、讲者备注。
- 优先支持“会议时把大纲按层级展开给别人看”。

### 5. 标签、链接与知识连接

本项目已有 `#标签` 和 `[[文档名]]` 的基础方向，适合构成本地知识库能力。

首版建议：

- 在节点文本和备注中识别 `#标签`。
- 按标签过滤文档和节点。
- 识别 `[[文档名]]` 作为弱链接。
- 在侧栏展示当前文档的链接列表。

后续建议：

- 文档名自动补全。
- 反向链接。
- 未创建文档提示。
- 标签全局页。
- 链接重命名跟随。

### 6. 搜索

搜索是本地优先产品的刚需，优先级高于复杂分类。

首版范围：

- 搜索文档标题、节点文本、备注。
- 支持标签过滤。
- 展示匹配文档列表。

后续范围：

- 命中节点高亮。
- 搜索结果定位到节点。
- 全文索引。
- 最近搜索和保存筛选。

### 7. 导入导出

幕布帮助中心强调从常见大纲/脑图格式导入，这是降低迁移成本的关键。本项目应优先支持开放格式，而不是绑定云服务。

当前适合支持：

- 导出文档：JSON、Markdown、OPML、FreeMind、HTML。
- 导出工作区：JSON。
- 导入文档：JSON、Markdown、OPML、FreeMind。
- 导入工作区：JSON。

后续可评估：

- XMind 和 MindManager 导入。
- PDF 或图片导出。
- ZIP 打包导出工作区和附件。
- 从幕布导出的 OPML/Markdown 迁入兼容性测试。

### 8. 分享与协作

幕布的在线分享和多人协作是云端产品的重要能力，但不适合直接进入本地优先首版。

本项目可替代实现：

- 用导出文件分享文档。
- 用 iCloud Drive 备份文件进行个人设备间迁移。
- 后续支持只读 HTML 导出，方便发给他人查看。

暂不做：

- 实时多人协作。
- 在线权限管理。
- 团队空间。
- 公网分享链接。
- 评论和协同编辑冲突处理。

## 主要用户工作流

### 工作流 1：快速记录和整理

1. 用户打开应用，进入最近编辑文档。
2. 使用 Enter 快速创建同级主题。
3. 使用 Tab / Shift+Tab 调整层级。
4. 折叠暂时不看的部分。
5. 自动保存到本地 IndexedDB。

成功标准：用户不需要先选择模板、保存路径或登录账号，就能开始记录。

### 工作流 2：从大纲切换脑图复盘

1. 用户完成一份结构化大纲。
2. 切换到脑图视图。
3. 选中关键节点检查结构是否平衡。
4. 回到大纲继续调整内容。

成功标准：切换视图不造成数据丢失，也不需要手动同步。

### 工作流 3：会议或汇报演示

1. 用户打开已有文档。
2. 需要只讲局部时，先聚焦某个节点。
3. 切换演示视图。
4. 按层级展示章节和要点。

成功标准：不替代专业 PPT，但足够支撑临时讨论、复盘和内部会议。

### 工作流 4：个人知识整理

1. 用户在节点中添加 `#项目`、`#读书`、`#待办` 等标签。
2. 用搜索或标签过滤找到相关文档。
3. 用 `[[文档名]]` 建立文档间关联。
4. 后续从反向链接或标签页继续扩展。

成功标准：知识连接轻量，不要求用户先学习复杂规则。

### 工作流 5：备份和迁移

1. 用户点击 iCloud 备份。
2. Electron 版本把工作区写入 iCloud Drive 的固定目录。
3. 浏览器版本在支持的情况下写入用户选择的目录，否则提示导出工作区。
4. 新设备可通过载入备份或导入工作区恢复。

成功标准：数据主权清晰，用户知道文件在哪里。

## 可替代实现范围

### 首版必须接近幕布的部分

- 大纲节点的快速编辑体验。
- 大纲和脑图同源。
- 层级调整、折叠、聚焦。
- 文档列表、搜索、标签。
- 常见格式导入导出。
- 本地自动保存。
- iCloud Drive 备份。

### 可以简化的部分

- 脑图样式和布局。
- 演示视图。
- 文档模板。
- 主题皮肤。
- 节点富文本。
- 附件系统。
- 分享页面。

### 不建议首版实现的部分

- 账号系统。
- 云端数据库同步。
- 多人实时协作。
- 团队权限。
- 支付和会员体系。
- 移动端原生应用。
- 插件市场。

## 产品取舍建议

1. 坚持本地优先
   - 默认不登录。
   - 默认不上传。
   - 明确告知数据保存在本机，iCloud 只是用户可控的备份位置。

2. 先把编辑器做顺
   - 大纲编辑体验比视觉装饰更重要。
   - 快捷键、焦点、撤销、拖拽、多选是后续质量提升重点。

3. 开放格式优先
   - JSON 保完整数据。
   - Markdown 适合阅读和二次编辑。
   - OPML 适合大纲工具互通。
   - FreeMind 适合脑图迁移。

4. 云协作留到更晚
   - 一旦做实时协作，就需要账号、权限、冲突解决、在线状态和服务端成本。
   - 当前产品定位更适合个人知识管理、写作、读书笔记和项目拆解。

## 后续开发优先级

P0：

- 稳定大纲编辑：撤销/重做、拖拽排序、多选。
- 完善导入导出兼容性。
- iCloud 备份恢复流程可视化。
- 文档搜索定位到具体节点。

P1：

- 脑图缩放、居中、导出图片。
- 标签全局页和反向链接。
- 回收站和历史快照。

P2：

- 附件和图片。
- 只读分享包或静态 HTML 发布。
- 原生菜单、系统快捷键、文件关联。
- 移动端或 iPad 版本调研。
